#  Metodología SCRUM

##  Enfoque

El desarrollo se realizó siguiendo un enfoque basado en Scrum, organizando el trabajo en iteraciones funcionales y priorizando funcionalidades.

---

##  Gestión de requerimientos

### 1. Recepción del requerimiento

- El requerimiento es validado y aprobado.
- Se incorpora al backlog del sistema.
- Se define el alcance inicial.
- Se planifica su revisión en una reunión tipo Sprint Planning.

---

### 2. Análisis y planificación

- Se revisa el requerimiento en detalle.
- Se identifican dependencias técnicas.
- Se analizan posibles riesgos.
- Se documentan dudas.
- Se estima el esfuerzo de desarrollo.
- Se definen tareas técnicas necesarias.

---

### 3. Planificación (Sprint Planning)

El trabajo se organizó en bloques funcionales:

- Proyectos
- Usuarios
- Tareas
- Kanban
- Soft delete
- Tiempo real

---

### 4. Desarrollo

- Se implementó cada funcionalidad de forma incremental.
- Se validó su correcto funcionamiento.
- Se mantuvo una arquitectura limpia.

---

### 5. Validación

- Se probaron los flujos principales.
- Se corrigieron errores antes de continuar.

---

##  Resultado

El uso de Scrum permitió:

- Organizar el desarrollo
- Priorizar funcionalidades
- Reducir errores
- Entregar valor progresivamente
